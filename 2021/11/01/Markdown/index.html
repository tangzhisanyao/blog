标题

要创建标题，请在标题文本前添加一至六个 # 符号。 您使用的 # 数量将决定标题的大小。

# 最大标题

## 二级标题

###### 最小标题

样式文本

样式 语法 输出
粗体 **这是粗体文本**  或 __这是粗体文本__ 这是粗体文本
斜体 *这是斜体文本* 或 _这是斜体文本_ 这是斜体文本
删除线 ~~这是错误文本~~ 这是错误文本
全部粗体和斜体 ***粗体和斜体结合*** 粗体和斜体结合

引用文本

可以使用> 来引用文本

引用代码

使用单反引号可标注句子中的代码或命令。 倒引号中的文本不会被格式化。

要将代码或文本格式化为各自的不同块，请使用三反引号。

链接

通过将链接文本包含在方括号 [ ] 内，然后将 URL 包含在括号 ( ) 内，可创建内联链接。

本站点是使用 GitHub Pages 构建的。

图像

您可以通过添加 ! 并在[ ]中包装 alt 文本来显示图像。 然后将图像链接包装在括号 () 中。

列表

无序列表

通过在一行或多行文本前面添加 - 或 * 可创建无序列表。

  • George Washington
  • John Adams
  • Thomas Jefferson

有序列表

要对列表排序，请在每行前面添加一个编号。

  1. James Madison
  2. James Monroe
  3. John Quincy Adams

嵌套列表

通过在一个列表项下面缩进一个或多个其他列表项，可创建嵌套列表。

  1. 第一个列表项
    • 第一个嵌套列表项
      • 第二个嵌套列表项

任务列表

要创建任务列表，在列表项目前面加一个常规空格字符，后接 [ ]。 要将任务标记为已完成，请使用 [x]。

  • #739
  • https://github.com/octo-org/octo-repo/issues/740
  • Add delight to the experience when all tasks are complete :tada:

表情符号

通过键入 :EMOJICODE: 可在您的写作中添加表情符号。

忽略Markdown格式

通过在 Markdown 字符前面输入 \，可告诉 GitHub 忽略（或规避）Markdown 格式。

表格

您可以用竖线 | 和横线 - 创建表格。 横线用于创建每列的标题，而竖线用于分隔每列。 必须在表格前包含空白链接，以便其正确呈现。
